Detailed Description

Immutable product identity constants for the application.

This header defines organization and application identity values that are compiled into the binary. These values are:

  • Read-only at runtime and constant for the lifetime of the process.
  • Intended for branding, About dialogs, log headers, and on-disk identifiers.
  • Safe to use from any thread (QStringLiteral creates static storage).

Scope: Identity only: names, domain, version, and stable folder identifiers.

Note
Do NOT place defaults/limits, computed paths, environment-derived values, or user-configurable settings here. Put those in npi::def instead.

Rationale: Centralizing identity avoids duplication and keeps UI, logs, and file layout consistent.

Build integration: Consider generating appVersion (and optionally a build channel or Git hash) via your build system (e.g., CMake configure_file) to keep it in sync.

Compatibility notes:
  • These constants are QStrings created with QStringLiteral. The literal data is stored in read-only program memory; construction is cheap and does not allocate on the heap.
  • Changing orgFolder or appFolder may change on-disk locations owned by the app.
