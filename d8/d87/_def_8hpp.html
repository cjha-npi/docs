Detailed Description

Shipped defaults and limits for the application.

This header defines compile-time defaults and operational limits that the application uses when no user or environment override is provided.

Scope
  • Tunable shipped defaults (filenames, sub-dirs, colors, timeouts).
  • Operational limits (max sizes, retry caps, rotation counts).
  • Format/encoding defaults (CSV delimiter, default charset).
  • Values are constant at runtime but may be changed per build.
Out of scope
  • Product identity (names, version, domains, GUIDs). See npi::iden.
  • Computed or environment-derived values (resolved paths, locale).
  • Secrets or machine-specific data.
Override / resolution order
The application should resolve settings using a clear precedence. A common pattern is: 1) Runtime config (e.g., QSettings, user config files) 2) Command line switches 3) Environment variables 4) Build-time defines (e.g., -DNPI_MAX_FILE_BYTES=...) 5) Shipped defaults in this header (npi::def)
Thread-safety
All symbols here are constants and are safe to read from any thread.
Build integration
  • Optionally expose selected defaults as compile definitions to allow per-build tuning without editing this file.
  • Consider documenting any changed defaults in release notes.
Compatibility notes
Changing defaults can affect user expectations and on-disk layout (e.g., subfolder names). Treat such changes as user-visible.
