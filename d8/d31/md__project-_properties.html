Project Properties

Introduction

This document describes the different project properties for a C/C++ project in Visual Studio that should be correctly to have projects working smoothly.

Solution and Project Names

When creating the solution/project always select different names for solution and project.

  • Always uncheck Place solution and project in the same directory when creating a solution/project.
  • Project folder is placed inside the solution folder
  • Convention is to name primary project as App

Language Standard

  • C++ Language Standard: C++20
  • C Language Standard: c17

Main Directories

The projects have a few important directories, by default these dikrectories are placed either inside the solution folder or inside the project folder. We move these directories to a different location for easier maintainence.

Output Directory:

This directory contains the generated .exe and .pdb. Projects inside the same solution should have the same output directory. By default it is <SolutionDir>\<Platform>\<Configuration> e.g. sol\x64\Debug. We change it to F:\Misc\VS_Bins\<SolutionName>\<Platform>_<Configuration>. To make this change happen, make sure the setting Project Properties > Configuration Properties > General > Output Directory is set to the following:

F:\Misc\VS_Bins\$(SolutionName)\$(Platform)_$(Configuration)\
Note
This folder is safe to delete, VS automatically creates this folder if it does not exist.

Intermediate Directory

This directory contains the intermediate files (e.g. .obj, ui_*.h, etc.). Each project should have its own intermediate directory so that files from different projects do not mingle. By default it is <SolutionDir>\<ProjectName>\<Platform>\<Configuration> e.g. sol\proj\x64\Debug. We change it to F:\Misc\VS_objs\<SolutionName>\<Platform>_<Configuration>_<ProjectName>. To make this change happen, make sure the setting Project Properties > Configuration Properties > General > Intermediate Directory is set to the following:

F:\Misc\VS_Objs\$(SolutionName)\$(Platform)_$(Configuration)_$(ProjectName)\
Note
This folder is safe to delete, VS automatically creates this folder if it does not exist.

Cache Location

This directory defines where the cache generated by Visual Studio is stored. Usually Visual Studio generates Many GBs of cache data and by default stores it in .vs file inside the solution folder. This makes solutions very bug in size and makes maintainence a bug haedache since (either delete all solution folders and lose all code files or manually open each solution folder and delete .vs file one by one). Changing it to an externam folder makes it much easier to free up space. To change it go to Visual Studio Options > Text Editor > C/C++ > Advanced Here under Browsing Database Fallback for Fallback Location select a folder, set Always Use Fallback Location to True and set Do Not Warn If Fallback Location Used (if available, in VS 2026 this option is no longer available) to True (otherwise it will warn every time a solution is opened). I am using the following folder currently

F:\Misc\VS_Caches`
Note
  • This has to be done only once. Since this is a Visual Studio settings, there is no need to do it per solution/project.
Warning
  • Do not delete the Fallback Location folder otherwise Visual Studio will complain and Intellisense will not work. To reclaim space open the folder and delete every thing inside it.
