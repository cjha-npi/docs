Show a fatal error box and terminate the process.

Displays an error dialog (OK + error icon). Adds MB_TOPMOST for better visibility in critical situations, then calls std::abort(). Intended for unrecoverable failures where continuing execution is unsafe.

Parameters
titleCaption text.
msgBody text.
hOptional owner HWND. Defaults to nullptr.
Postcondition
The function does not return; the process is aborted.
Warning
  • If you pass an HWND that belongs to a QWidget, ensure you did not obtain it on a worker thread. Do not call QWidget APIs off the GUI thread.
  • Use only for fatal conditions. Consider logging to persistent storage before calling this function if possible.

void npi::win::msgBox ( const QString & title,
const QString & msg,
const HWND h,
const UINT flags )
inlinenoexcept

Show a Windows message box with custom flags.

This is the base helper around MessageBoxW. It adds sane defaults: if no modality flag is supplied, it uses MB_APPLMODAL when an owner is present and MB_TASKMODAL when there is no owner. When there is no owner, MB_SETFOREGROUND is added to reduce z-order issues.

Parameters
titleCaption text (UTF-16 via QString conversion).
msgBody text (UTF-16 via QString conversion).
hOwner window handle. May be nullptr. If obtained from QWidget, it must have been retrieved on the GUI thread.
flagsStandard MessageBox flags (buttons, icon, modality, etc.).
Note
Callable from any thread. Does not interact with QWidget.
Warning
If you pass an HWND that belongs to a QWidget, ensure you did not obtain it on a worker thread. Do not call QWidget APIs off the GUI thread.
See also
MessageBoxW
