Detailed Description

Windows-specific utilities and wrappers.

The namespace contains helpers that directly interact with the Windows API. Grouping such functionality here keeps platform- specific code localized and explicit.

Show an error message box and then terminate the process.

Displays a fatal error dialog using MB_OK | MB_ICONERROR and then calls std::abort() unconditionally. Use this only for unrecoverable conditions. No cleanup handlers are run after std::abort().

void npi::win::msgBox ( const QWidget * wid,
const QString & title,
const QString & msg,
UINT flags )
inlinenoexcept

Show a Windows message box, parented to an optional QWidget.

Converts the caption and body to UTF-16 and invokes MessageBoxW. If an owner widget is provided, its top-level window becomes the owner. If no owner is available, the function strengthens modality to task-modal and requests foreground activation to reduce the risk of the dialog appearing behind other windows.

Modality policy:

  • If the caller does not supply any MB_* modality flags, the function applies MB_APPLMODAL when an owner HWND exists, otherwise MB_TASKMODAL.
  • If the caller supplies a modality flag, it is honored as-is.
Parameters
widOptional QWidget whose top-level window will own the dialog. May be nullptr; child widgets are accepted.
titleCaption text (QString).
msgBody text (QString).
flagsCombination of MB_* flags (icons, buttons, modality, etc.).
Remarks
  • Calling winId() can materialize a native window for the owner, which is typically desired for correct parenting and Z-order behavior.
  • The function does not return a button ID. If the result is needed, introduce an overload that returns the int value of MessageBoxW.
