Detailed Description

Header defining RefPtr classes with polymorphic support.

This file contains the definition of RefPtr classes. The RefPtr class is a smart pointer that provides non-owing handles to objects managed by RefPtrHost with automatic nullifying on teardown via a shared atomic pointer.

Note
  • This implementation is inspired by QPointer and std::shared_ptr/std::weak_ptr.
  • This is a header-only implementation because we are using templates.
  • RefPtr: Non-owning, auto-nullifying pointer-like handle to an object of type T.
  • RefPtrHost: Owning host (shared) for an object of type T, publishes observers.
  • RefPtrWeakHost: Weak reference to a RefPtrHost and its observers.
  • RefPtrFromThis: Enable-from-this helper for classes wrapped in RefPtrHost.

RefPtr: Provides a QPointer-like handle to an object managed by RefPtrHost.

  • Non-Owning: Does not own the object T and does not manage or extend its lifetime.
  • Auto-nullifying: Automatically nullifies when the host is destroyed.
  • Thread-Safe: Safe to use in multithreaded environments.
  • Safe boolean conversion: Provides a safe boolean conversion to check if the linked object is still alive.
  • Direct pointer-like access: Provides direct pointer-like access to the object.
  • Polymorphic up-/down-cast: Supports polymorphic conversion and comparison.

RefPtrHost: Provides a std::shared_ptr-like wrapper for an object of type T.

  • Owning: Owns the object T and destroys it when the last reference is released.
  • Custom Destructor: Allows optional custom destructor for the object T.
  • Thread-Safe: Safe to use in multithreaded environments.
  • Safe boolean conversion: Provides a safe boolean conversion to check if the underlying object T is still alive.
  • Dual Observers: Maintains two observer channels for getting RefPtr objects:
  • Universal Channel: RefPtr object obtained through this channel is valid while the underlying object T is alive.
  • Local Channel: RefPtr object obtained through this channel is only valid till last local copy of RefPtrHost is alive.
  • Direct pointer-like access: Provides direct pointer-like access to the object.
  • Polymorphic up-/down-cast: Supports polymorphic conversion and comparison.

RefPtrWeakHost: Provides a std::weak_ptr-like wrapper for an object of type T.

  • Non-Owning: Does not own the object T and does not manage or extend its lifetime.
  • Thread-Safe: Safe to use in multithreaded environments.
  • Weak Reference: Provides a weak reference to the RefPtrHost object's underlying object T and its Observers.
  • Locking: Provides a method to lock the weak reference and obtain a RefPtrHost object if the underlying object T and observers (Universal and Local) are still valid.
  • Expired Check: Provides a method to check if the underlying object T and/or observers (Universal and/or Local) are expired.

RefPtrFromThis: Provides a std::enable_shared_from_this-like mechanism for an object of type T.

  • Inheritance: This class is supposed to be inherited by any class that is to be wrapped in RefPtrHost.
  • Enables RefPtr creation: Allows an object of type T to create a RefPtr to itself when wrapped in RefPtrHost.
  • Universal RefPtr: Provides a method to create a RefPtr object using the RefPtrHost's universal observer.

Thread-safety model

  • Safety against use-after-free: When observers are destroyed, they store nullptr into a shared atomic<void*>. Readers load that pointer atomically and will observe nullptr on teardown. This avoids dereferencing a freed object.
  • Data visibility for T itself: The acquire on the atomic pointer only pairs with the release store of nullptr in Obs::~Obs. It does NOT provide general visibility for arbitrary writes to fields of T. Concurrent reads/writes of T must be synchronized by the caller (e.g. external mutexes or lock-free discipline).

Usage

// Create a host for MyClass
npi::RefPtrHost<MyClass> host(new MyClass());
// Universal RefPtr: valid while the hosted object is alive (any host copy)
npi::RefPtr<MyClass> uni = host.universal_();
if (uni) uni->doSomething_();
// Local RefPtr: valid only while a host copy that shares this "local observer" is alive
npi::RefPtr<MyClass> loc = host.local_();
if (loc) loc->doSomethingElse_();
// Weak host: test expired or lock to a new RefPtrHost
npi::RefPtrWeakHost<MyClass> weak = host.weakHost_();
if (auto locked = weak.lock_(); locked) {
locked->doSomething_();
}
// Enable-from-this:
struct Self : npi::RefPtrFromThis<Self> { void hello_(); };
auto selfHost = npi::RefPtrHost<Self>(new Self());
auto selfRefPtr = selfHost.universal_()->refPtrFromThis_(); // or selfRefPtr = self->refPtrFromThis_()
if (selfHost) selfHost->hello_();

Implementation details

Internally, RefPtrHost<T> owns T in a std::shared_ptr<T> (obj_). It also holds two observers (uni_ and loc_), which are instances of the inner Obs struct.

Each Obs struct contains a std::shared_ptr<std::atomic<void*>>. This atomic<void*> holds the type-erased pointer to T.

A RefPtr<T> simply holds a std::shared_ptr to that same std::atomic<void*>.

When the last shared_ptr<Obs> is destroyed (e.g., when loc_ goes out of scope), Obs::~Obs is called. The destructor stores nullptr into the atomic<void*> using std::memory_order_release.

When RefPtr<T>::get_() is called, it loads the pointer from the atomic<void*> using std::memory_order_acquire. This acquire/release pairing guarantees that any RefPtr will observe nullptr once its corresponding observer is destroyed, preventing use-after-free.

  • Universal channel: universal_() returns a RefPtr tied to uni_. uni_ is shared by all copies of the RefPtrHost, so this RefPtr stays valid as long as obj_ is alive.
  • Local channel: local_() returns a RefPtr tied to loc_. loc_ is not always shared (e.g., duplicateWithNewLocal_() creates a new one). This RefPtr only stays valid as long as a RefPtrHost sharing that specific loc_ observer exists.

The RefPtrWeakHost holds std::weak_ptr to obj_, uni_, and loc_, allowing it to be "locked" back into a full RefPtrHost if all three are still alive.

The RefPtrFromThis class holds a std::weak_ptr to uni_, allowing an object to create a RefPtr to itself.
