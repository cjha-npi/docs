Conventions

Introduction

The Conventions page defines the common rules and conventions used throughout the project. These include naming, commenting, source organization, folder structure, documentation practices, and related design choices. The goal is to ensure consistency across the codebase and project material so that development, maintenance, and collaboration remain predictable and manageable.

Project Structure

inc/npi/ -> Public headers under the npi namespace
inc/ext/ -> External headers
src/ -> Source files
gui/ -> GUI-related resources
img/ -> Images
res/ -> Resources

Coding Conventions

  • Everything has to be inside either named or anonymous namespace.
  • NO single word simple names e.g. Font, Show, etc. for Class and Struct.
  • All member variables and member functions end with an underscore _.
  • Static members and static functions start with a capital letter.

Comment Conventions

  • ASCII-only (no Unicode symbols or smart quotes)
  • Use \brief and \details for Doxygen descriptions
  • Use /** ... */ for files, classes, structs, etc.
  • Use /// for functions
  • Use ///< for single-line member documentation
  • No Doxygen comments inside function bodies
