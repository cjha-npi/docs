Detailed Description

template<typename T>
class npi::LinkWeakHost< T >

A non-owning weak host that references a LinkHost-managed object and its observers including local observer.

  • Behaves like a std::weak_ptr i.e. holds a weak reference to the object and observers.
  • Allows lock() to temporarily acquire a strong LinkHost if the object is still alive.
  • Provides polymorphic comparisons.
Template Parameters
TThe type of the object being weakly hosted. T must be wrapped in LinkHost.
Note
LinkWeakHost also contains a weak reference to the local observer in the LinkHost. This means that if the local observer is destroyed, the LinkWeakHost will not be able to create a valid LinkHost object using its lock_() method and will return true for expired_().
Example of using LinkWeakHost:
LinkHost<MyClass> host(new MyClass());
LinkWeakHost<MyClass> weak(host);
if (auto locked = weak.lock_()) {
locked->doSomething_();
}

◆ lock_()

template<typename T>
LinkHost< T > npi::LinkWeakHost< T >::lock_ ( ) const
inlinenoexcept

This function attempts to create a new LinkHost object from this LinkWeakHost object if the underlying object is still alive. This method calls lock() on the weak pointers to obtain the shared pointers to the object T as well as universal and local observers from which it was created. If any of the lock() fails the returned LinkHost object will be empty.

Returns
A LinkHost object that is an exact copy of the LinkHost object from which this LinkWeakHost was created. If the object is no longer alive, the returned object will be empty.
Note
Locking only succeeds if all the three std::shared_ptr: obj_, uni_, and loc_ inside the LinkHost object are still alive.
