Detailed Description

Common namespace for application data and utilities.

This namespace contains module-agnostic application-wide identifiers, compile-time defaults, and lightweight utilities.

Note
Only for cross-cutting code; place domain-specific code in a dedicated child namespace e.g. npi::log or npi::net.

const QString & npi::app::appProgramDataDir ( )
nodiscardnoexcept

Absolute application-level ProgramData path.

Returns the absolute, normalized path to: [ProgramData]/[OrganizationName]/[ApplicationName] Implementation:

  • Obtains the organization-level base from orgProgramDataDir().
  • Ensures the application subdirectory exists (creates it; fatal if it fails).
  • Canonicalizes when possible; otherwise cleans the absolute path.
  • Converts separators to forward slashes; UNC is returned as //server/share/....

The result is cached in a function-local static and reused on subsequent calls.

Returns
Reference to a process-lifetime QString with the absolute path using forward slashes.
Postcondition
The returned reference remains valid for the lifetime of the process.
Note
  • Windows-only: Paths use forward slashes for Qt friendliness; call QDir::toNativeSeparators(...) before passing to Win32 APIs.
  • Multi-Thread: Safe, first-call initialization is thread-safe (C++11). Subsequent calls are lock-free.
Warning
On unrecoverable failure to create the directory, a native dialog is shown and the process terminates (no return).
Example
const QString appPD = npi::app::appProgramDataDir(); // e.g. C:/ProgramData/npi_electronic/test_app
const QString cfg = QDir(appPD).filePath("settings.ini");

◆ orgProgramDataDir()

const QString & npi::app::orgProgramDataDir ( )
nodiscardnoexcept

Absolute organization-level ProgramData path.

Returns the absolute, normalized path to: [ProgramData]/[OrganizationName] Resolution order: 1) Windows API (SHGetKnownFolderPath with FOLDERID_ProgramData) 2) Environment variable fallback: PROGRAMDATA Then:

  • Verifies the base folder exists (fatal if missing).
  • Ensures the organization subdirectory exists (creates it; fatal if it fails).
  • Canonicalizes when possible (resolves junctions/symlinks); otherwise cleans the absolute path.
  • Converts separators to forward slashes; UNC is returned as //server/share/....

The result is cached in a function-local static and reused on subsequent calls.

Returns
Reference to a process-lifetime QString with the absolute path using forward slashes.
Postcondition
The returned reference remains valid for the lifetime of the process.
Note
  • Windows-only: Paths use forward slashes for Qt friendliness; call QDir::toNativeSeparators(...) before passing to Win32 APIs.
  • Multi-Thread: Safe, first-call initialization is thread-safe (C++11). Subsequent calls are lock-free.
Warning
On unrecoverable failure (cannot locate or create the required directory), a native dialog is shown and the process is terminated (no return).
Example
// Build a file path within the org folder (Qt accepts forward slashes)
const QString orgDir = npi::app::orgProgramDataDir(); // e.g. C:/ProgramData/npi_electronic
const QString file = QDir(orgDir).filePath("global.log");
// When calling Win32, convert once:
const QString native = QDir::toNativeSeparators(orgDir); // e.g. C:\ProgramData\npi_electronic
// CreateDirectoryW(reinterpret_cast<LPCWSTR>(native.utf16()), nullptr);
