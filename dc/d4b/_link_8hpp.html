Detailed Description

Header-only Link family: Link, LinkHost, LinkWeakHost, LinkFromThis.

Overview

Link: Provides a QPointer-like handle to an object managed by LinkHost.

  • Non-Owning: Does not own the object T and does not manage or extend its lifetime.
  • Auto-nullifying: Automatically nullifies when the host is destroyed.
  • Thread-Safe: Safe to use in multithreaded environments.
  • Safe boolean conversion: Provides a safe boolean conversion to check if the linked object is still alive.
  • Direct pointer-like access: Provides direct pointer-like access to the object.
  • Polymorphic up-/down-cast: Supports polymorphic conversion and comparison.

LinkHost: Provides a std::shared_ptr-like wrapper for an object of type T.

  • Owning: Owns the object T and destroys it when the last reference is released.
  • Custom Destructor: Allows optional custom destructor for the object T.
  • Thread-Safe: Safe to use in multithreaded environments.
  • Safe boolean conversion: Provides a safe boolean conversion to check if the underlying object T is still alive.
  • Dual Observers: Maintains two observer channels for getting Link objects:
    • Universal Channel: Link object obtained through this channel is valid while the underlying object T is alive.
    • Local Channel: Link object obtained through this channel is only valid till last local copy of LinkHost is alive.
  • Direct pointer-like access: Provides direct pointer-like access to the object.
  • Polymorphic up-/down-cast: Supports polymorphic conversion and comparison.

LinkWeakHost: Provides a std::weak_ptr-like wrapper for an object of type T.

  • Non-Owning: Does not own the object T and does not manage or extend its lifetime.
  • Thread-Safe: Safe to use in multithreaded environments.
  • Weak Reference: Provides a weak reference to the LinkHost object's underlying object T and its Observers.
  • Locking: Provides a method to lock the weak reference and obtain a LinkHost object if the underlying object T and observers (Universal and Local) are still valid.
  • Expired Check: Provides a method to check if the underlying object T and/or observers (Universal and/or Local) are expired.

LinkFromThis: Provides a std::enable_shared_from_this-like mechanism for an object of type T.

  • Inheritance: This class is supposed to be inherited by any class that is to be wrapped in LinkHost.
  • Enables Link creation: Allows an object of type T to create a Link to itself when wrapped in LinkHost.
  • Universal Link: Provides a method to create a Link object using the LinkHost's universal observer.

Usage

// Create a host for MyClass
auto host = npi::LinkHost(new MyClass());
// Universal link: valid until host (and all copies) are destroyed
npi::Link uni = host.universal_();
if (uni) uni->doSomething();
// Local link: valid only while this specific host (or its duplicates) remain alive
npi::Link loc = host.local_();
if (loc) loc->doSomethingElse();
// Weak host: can test expired or lock to a new LinkHost
npi::LinkWeakHost weak = host.weakHost_();
if (auto locked = weak.lock_(); locked) {
locked->doSomething();
}
// Enable-from-this:
struct Self : npi::LinkFromThis<Self> { void hello(); };
Self* self = new Self();
auto selfHost = npi::LinkHost(self);
auto selfLink = self->linkFromThis_();
if (selfLink) selfLink->hello();

Implementation Details

Internally, LinkHost<T> holds a std::shared_ptr<T> that manages the object T. It has a Obs struct which has a std::shared_ptr<std::atomic<void*>> ptr_ which is used to store a type-erased reference to the object T. The Link<T> class holds a std::shared_ptr<std::atomic<void*>> ptr_ which is the same as the ptr_ in the Obs struct. This way Obs struct provides a way for Link objects to access the underlying main object T. The use of type-erased reference allows for polymorphic behavior. In the destructor of the Obs struct, the ptr_ is replaced with a null pointer, effectively nullifying all Link objects that are linked to it. This is done in a thread-safe manner using std::memory_order_release.

The LinkHost class has two objects of std::shared_ptr<Obs>: uni_ and loc_. The uni_ is shared among all LinkHost copies, but the loc_ is created as new whenever the LinkHost is copied using duplicateWithNewLocal_() method. During normal copying both uni_ and loc_ are copied. The Link object created using the universal_() method is linked to the uni_ object and will remain valid as long as any copy of the LinkHost object is alive anywhere in the application. While the Link object created using the local_() method is linked to the loc_ object and will remain valid only as long as at least one copy of the LinkHost object created using duplicateWithNewLocal_() is alive.

The Link object's get_() method atomically loads the stored pointer using std::memory_order_acquire to synchronize with any release stores that may have updated the pointer elsewhere. This ensures that all prior writes to the object are visible to the calling thread. It then uses static_cast to convert the void* to the correct object type and returns the resulting T*. The Link and LinkHost classes also provides a reset_() method to release ownership by calling reset() on the internal std::shared_ptr's.

The LinkWeakHost class contains three std::weak_ptr objects that point to the obj_, uni_ and loc_ objects of the LinkHost class. It provides a method lock_() that tries to lock all three weak pointers and if all three locks succeed then it returns a LinkHost object created using these three std::shared_ptr of obj_, uni_ and loc_. If any of the locks fails then it returns a LinkHost object that has all three pointers as null. The expired_() method checks if any of the three weak pointers are expired or not.

The LinkFromThis class has a mutable std::weak_ptr<LinkHost::Obs> weakUni_; which is used to store a weak pointer to the uni_ object of the LinkHost class when the object is wrapped in LinkHost. It provides a method linkFromThis_() which returns a Link object created using the uni_ object of the LinkHost class. It also provides a method isLinkFromThisAvailable_() which returns true if the call to linkFromThis_() can be made and false otherwise. This is done by checking if the weakUni_ object is expired or not.
