Windows-specific routine for resolving paths.

This anonymous namespace contains Windows-specific routine for resolving Known Folders to absolute paths as QString values.

QString anonymous_namespace{AppUtil.cpp}::knownFolderPathWin ( REFKNOWNFOLDERID id)
noexcept

Retrieve a Windows Known Folder absolute path as a QString.

Parameters
idThe REFKNOWNFOLDERID of the requested folder (e.g. FOLDERID_ProgramData, FOLDERID_PublicDocuments).
Returns
Absolute path on success; an empty QString on failure (including an HRESULT failure from SHGetKnownFolderPath or a null pointer result).

Invokes SHGetKnownFolderPath with KF_FLAG_DEFAULT to obtain the filesystem path for the specified Known Folder. On success, the shell allocates a wide-character buffer; this function converts it to QString and then releases the buffer via CoTaskMemFree. No exceptions are thrown; errors are reported by returning an empty QString.

Note
  • Available only on Windows builds (_WIN32).
  • The returned QString is a value; callers incur normal implicit-sharing refcount costs.
  • The function is noexcept: it does not throw exceptions. Callers are responsible for checking for an empty QString result and handling errors.
Error handling
  • If SHGetKnownFolderPath returns a failure HRESULT (FAILED(hr)) or a null output pointer, this function returns an empty QString.
Thread safety
  • The function is thread-safe: it uses no shared mutable state.
See also
SHGetKnownFolderPath, FOLDERID_ProgramData, FOLDERID_PublicDocuments
