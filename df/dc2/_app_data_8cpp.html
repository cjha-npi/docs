Detailed Description

Windows implementations for C:\ProgramData and C:\Users\Public\Documents resolution.

Resolution order per location: 1) Windows Known Folders (SHGetKnownFolderPath):

  • FOLDERID_ProgramData
  • FOLDERID_PublicDocuments 2) Fallback environment variables:
  • PROGRAMDATA
  • PUBLIC

Creation rules:

  • Only the organization and application subfolders are created via QDir::mkpath().
  • The system folder C:\ProgramData or C:\Users\Public\Documents is NOT created if it is missing; return empty instead.

Error handling:

  • No exceptions are thrown. An empty QString indicates failure.
  • Callers MUST check for empty and treat as a fatal startup error.

Thread-safety and performance:

  • Each function caches its result in a function-local static; initialization is thread-safe in C++11 and later. Subsequent calls are O(1).

Build and link notes (Windows):

  • Required headers: <windows.h>, <knownfolders.h>, <shlobj.h>
  • Link with: Shell32 and Ole32
    • MSVC/clang-cl: #pragma comment(lib, "Shell32.lib"), #pragma comment(lib, "Ole32.lib")
    • CMake: target_link_libraries(your_target PRIVATE Shell32 Ole32)
    • MinGW: -lshell32 -lole32
