Detailed Description

template<typename T>
class npi::Link< T >

A thread-safe, non-owning, auto-nullifying safe pointer to an object of type T.

  • Does not own the object T.
  • Object must be wrapped in LinkHost.
  • Provides safe pointer-like access to T.
  • Supports polymorphic conversion and comparison.
  • Offers a safe boolean conversion to check if the linked object is still alive.
Template Parameters
TThe type of the object being linked to.
Example of using the explicit boolean operator:
Link<A> link;
// assignment from LinkHost<A>
if(link) {
link->hello_(); // calling base class method
}

template<typename T>
T * npi::Link< T >::get_ ( ) const
inlinenodiscardnoexcept

Retrieve a raw pointer to the managed object, if any.

Returns
A pointer to the object of type T, or nullptr if no object is set.
Warning
Check for validity before using the pointer to the object.
Note
  • This method is marked noexcept, so it guarantees not to throw.
  • It is also marked [[nodiscard]] to encourage callers to check for nullptr rather than silently ignoring the return value.
  • Because it performs an atomic load, it is safe to call concurrently from multiple threads (readers).

This method returns the underlying raw pointer of type T* that is stored inside an atomic wrapper within a std::shared_ptr. It performs the following steps:

  1. Tests whether the internal std::shared_ptr<std::atomic<void*>> ptr_ is non-null.
    • If ptr_ is null, this method immediately returns nullptr.
  2. If ptr_ is non-null, loads the stored void pointer atomically using std::memory_order_acquire to synchronize with any release stores that may have updated the pointer elsewhere. This ensures that all prior writes to the object are visible to the calling thread.
  3. Uses static_cast<T*> to convert the void* to the correct object type and returns the resulting T*
