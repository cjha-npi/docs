Detailed Description

Native Windows message box helpers for emergency or low-level use.

This header provides thin wrappers around MessageBoxW intended for cases where Qt UI may be unavailable or unreachable (e.g., very early startup, very late shutdown, logging failure paths, or when the caller is not on the GUI thread and cannot safely touch QWidget).

The functions are callable from any thread. When an owner HWND is provided, Windows will place the message box relative to that window. When no owner is provided, the helpers select reasonable defaults for modality and foreground activation to improve visibility.

Integration guidance:

  • Prefer QMessageBox on the GUI thread for normal UX.
  • Use these helpers as a fallback when qApp is not available or when touching QWidget is not safe.
Warning
If the HWND originates from a QWidget, obtain it on the GUI thread (e.g., via winId()) and pass only the raw HWND to these functions.

Threading:

  • These helpers do not access QWidget or other Qt GUI types.
  • Calling MessageBoxW from worker threads is supported by Win32.

Error handling:

  • All functions are declared noexcept; failures inside QString conversions would result in process termination as per noexcept rules. This is acceptable for the intended "last resort" usage.
