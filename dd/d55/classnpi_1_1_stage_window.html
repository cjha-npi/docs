Detailed Description

Helper to display a top-level widget with optional animation and robust on-screen positioning.

This class provides a convenient way to show a top-level widget with various display options, including immediate display and animated transitions. It handles positioning the widget within the available screen space and offers customization options for the animation duration and easing curve.

void npi::StageWindow::animate_ ( ) const
inline

Show with an animated transition from a small source rect (cursor or caller) to the target.

Steps:

  1. Compute begRect from caller_ or a tiny rect at QCursor::pos().
  2. Call positionWidget_() to place within screen bounds and center if requested.
  3. Set target widget opacity to 0.0 and show it using callWidgetCorrectShowMethod_().
  4. Create a temporary frameless tool window (overlay) at begRect with a background color derived from the target widget palette (slightly lightened/darkened for contrast).
  5. Animate the overlay geometry from begRect to the target frame geometry.
  6. On finish, set target opacity to 1.0 if still alive and delete the overlay.
    Note
  • The overlay is Qt::Tool and WA_TransparentForMouseEvents to avoid focus/taskbar issues.
  • The animation object uses DeleteWhenStopped for cleanup.
  • The lambda does not capture this to avoid lifetime hazards.

void npi::StageWindow::positionWidget_ ( ) const
inlineprivate

Compute and apply an on-screen position for widget_, respecting frames/title bars.

Logic:

  • Estimate full framed size if not yet shown (geometry == frameGeometry).
  • If center_ is set, center over it (using frame geometry when it is a window).
  • Else, if the widget has never been displayed and sits at (0,0), center on the screen under the cursor.
  • Select screen using the frame-geometry center of widget_ (or center_); fallback to primary.
  • Clamp position to available geometry (no overflow left/top/right/bottom).
