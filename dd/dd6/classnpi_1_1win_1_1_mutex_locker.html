Detailed Description

RAII wrapper around a Windows named mutex.

Constructs or opens a named mutex, waits to acquire ownership, and releases it on destruction. The name is qualified with the "Local\\" prefix to scope the mutex to the current session. This class therefore acts as a lock between:

  • Multiple threads of the same process, and
  • Multiple processes/applications in the same Windows logon session.

Key points:

  • Constructor blocks until the mutex is acquired, unless CreateMutexW fails.
  • Destructor safely releases and closes the handle if owned/opened.
  • Copy and move are disabled to prevent double-release issues.
  • WAIT_ABANDONED is treated as successful acquisition as per Win32 semantics.

npi::win::MutexLocker::MutexLocker ( const QString & name)
inlineexplicit

Create or open a session-local named mutex and acquire it.

Parameters
nameLogical mutex name (without "Local\\"). Example: "npi-SharedFileLock".

Internally, the name is converted to UTF-16 and prefixed with "Local\\". This constructor: 1) Calls CreateMutexW(nullptr, FALSE, L"Local\{name}"). Passing FALSE means the mutex is not acquired at creation time. 2) If the handle is valid, calls WaitForSingleObject(h, INFINITE). On WAIT_OBJECT_0 or WAIT_ABANDONED, the lock is considered acquired. Otherwise, the handle is closed and the locker remains invalid. If another thread/process already owns the mutex, this call blocks until it becomes available.

◆ unlock_()

void npi::win::MutexLocker::unlock_ ( )
inline

Release the mutex if currently owned by this instance.

Calls ReleaseMutex(handle_) only when locked_ == true. After a successful release, sets locked_ to false. It does not close the HANDLE; the destructor will close the HANDLE. Note: ReleaseMutex must be called exactly once for each successful acquisition on the same thread. This class ensures that by tracking locked_.
