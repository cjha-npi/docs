Detailed Description

Common utility helpers used across the application.

This header declares small, reusable helper functions. Keep utilities focused, side effect free when possible, and independent of global state. Prefer thin, well-named helpers over large frameworks.

Scope
  • String helpers (e.g., trim, case-insensitive compare, hex encode).
  • Filesystem helpers (safe path join, temp file naming, existence checks).
  • Time and numeric helpers (duration parsing, clamping, unit conversions).
  • OS/Win32 shims where needed, wrapped with clear error handling.
Design guidelines
  • Prefer view types for inputs: QStringView / QAnyStringView.
  • Return owning types for outputs where appropriate: QString / QByteArray.
  • Keep headers light: include only what the declarations need.
  • Inline only trivial helpers; put non-trivial code in Util.cpp.
Note
No hidden static state. All functions must be thread-safe.
Error handling
  • Return bool with out-parameter, or return optional-like results.
  • Do not log inside generic helpers; let callers decide how to report.
Organization
  • If this file grows, split by domain under inc/npi/util/ (e.g., Str.hpp, Fs.hpp, Win.hpp) and keep this header as a small facade.
Naming
  • Free functions and non-member symbols do not use a trailing underscore (project convention). Reserve the trailing underscore for class members.
Dependencies
  • Prefer Qt for app-facing utilities; keep pure algorithmic helpers free of Qt and place them in a separate header if needed.
