Detailed Description

RAII locker for a Windows named mutex for both inter-process and inter-thread synchronization.

This header defines npi::GlobalLocker, a small RAII helper that acquires and releases a Windows named mutex using CreateMutexW/WaitForSingleObject/ReleaseMutex. Because it uses a named mutex, it can synchronize:

  • Threads within the same process (intra-process), and
  • Different processes/applications (inter-process) running on the same machine/session.

Scope and visibility:

  • The constructor prepends "Local\\" to the provided name. This limits visibility to the current logon session. Use "Global\" (not provided here) for a machine-wide namespace requiring appropriate privileges and Terminal Services mutex namespace support.

Ownership and lifetime:

  • On success, the constructor blocks until the mutex can be acquired. Destruction releases the mutex and closes the HANDLE.
  • ReleaseMutex is called only if this instance actually holds the lock.
  • CloseHandle is always called if a HANDLE was created to avoid handle leaks.

WAIT_ABANDONED handling:

  • If a previous owner terminated without releasing the mutex, WaitForSingleObject returns WAIT_ABANDONED. This class treats it as a successful acquisition (Windows semantics) and sets locked_ = true. Validate any shared state guarded by the mutex if needed.
